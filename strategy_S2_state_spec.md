# S2 需求规格:状态表示与统一协议(策略 AI 的输入层)【s2.v4 修订版】

> 策略 AI 项目第二阶段。S1 引擎已完成并验证(算分正确、胡牌主链路通、零和守恒)。
> 本文档是 **S2 唯一执行规格**(v4,全面替代 v3),已吸收 S1 全部规则:庄家轮转、杠转水、一炮多响、杠完整记录、流局退杠钱、海底禁杠、连续杠、过庄锁定、抢杠胡。
> Python 3.10+,依赖少,每模块有单测。

## v4 修订说明(为什么重写)

**最终产品定位已明确:现实世界麻将辅助模型。** 部署时的输入来自视觉识别与人工录入,存在三类固有缺陷:

1. **中途接入**:模型可能在牌局进行到一半时才开始观测,换三张、前序弃牌历史等信息不可知;
2. **观测残缺**:换三张流向、巡数、牌墙精确剩余等字段随时可能缺失;
3. **观测噪声**:视觉识别不保证 100% 准确,弃牌/副露的识别结果带有不确定性。

v3 的隐含假设是"观测完整、精确、从第 0 手开始连续记录",与部署现实矛盾。若按 v3 训练,模型只会在完美观测下打牌,接入真实噪声输入时性能会断崖下跌(sim2real gap)。

**v4 的核心转向:S2 不是"完美信息编码器",而是"残缺信息下的决策输入层"。**

相对 v3 的主要变化:

| 变化 | 内容 |
|---|---|
| 新增 | 观测状态成为一等公民:每个事实字段带 `observed / estimated / unknown` 标记与置信度 |
| 新增 | 观测退化模块 `observation_degradation.py`:训练时把完美观测故意弄残缺(遮换三张、截历史、注入识别噪声),弥合训练与部署的分布差距 |
| 新增 | 合法动作判定抽成共享纯函数 `legality.py`:训练与部署同源,部署时不依赖 S1 引擎在场 |
| 新增 | 视觉事件流 schema 现在定稿(不再"预留"),含置信度与 108 张守恒调和 |
| 新增 | 规则配置(封顶番/自摸模式)进入观测,模型可条件化于不同玩法 |
| 重构 | `tile_inference.py` 拆为 `tile_counting.py`(确定性计数,硬编码,上帝视角对账验收)与 `tile_belief.py`(学习式信念,上帝视角做监督标签) |
| 挪位 | 已胡玩家亮出的手牌:从推理层"反推"挪入**事实层** `seen_counts`(亮牌是确定事实,是尾盘余牌精度的最大来源) |
| 砍掉 | 手写的行为学启发式:断章分析、花色兴趣分数、攻防倾向、胡牌路线反推等(`fragment_inference.py`、`opponent_model.py` 整体移除)。原因:无验收 oracle、拍脑袋参数会注入系统性偏差、且封死模型上限。对手建模能力改由学习式 `tile_belief` 承担;断章类直觉可作为 S3 规则基线的内部启发式,不进入 S2 观测 |
| 保留 | 三层划分、可见性红线、动作空间、相对位置编码、引擎→协议→编码的整体架构 |

**S1 引擎不受影响**,且价值增大:它既是训练环境与裁判,又新增一个合法用途——**上帝视角作为 belief 学习的监督标签**(只用于批改,严禁作为模型输入)。

## 这一步在整条线里的位置

```
S1 引擎(已完成):能打牌、能算分、是环境+裁判+标签工厂
  ↓
[S2 状态表示与协议] ← 当前
  ├─ 统一状态协议(JSON):引擎/视觉/人工录入 三种来源都转成它,且允许字段"未知"
  ├─ 观测退化:把引擎的完美观测退化成"像真实部署一样残缺"的训练样本
  ├─ 把(可能残缺的)状态编码成定长张量(模型的"输入")
  ├─ 把动作编码成索引(模型的"输出"),合法性判定训练/部署同源
  └─ 确定性计数硬编码 + 学习式信念接口
  ↓
S3 规则AI(用统计层特征做启发式)→ S4 监督学习 → S5 强化学习(必须在退化观测上训练)→ 部署(视觉输入)
```

**核心判据(贯穿全文的分工原则)**:

> **一个量若存在唯一正确答案、能写出验收测试(有 oracle)→ 硬编码,程序做到满分;
> 若只能靠揣测对手行为、无解析公式 → 学习式,用上帝视角做标签训练,绝不手写公式。**

## 一、三层信息模型(v4 重定义)

设当前决策玩家为"自己",其余按下家/对家/上家相对位置编号(相对位置,不是绝对座位,便于模型泛化)。

### 第 0 原则:可观测性标记(v4 新增,地基)

协议中**每个事实字段**都携带观测状态:

- `observed`:直接观测到或用户确认的事实(置信度 1.0);
- `estimated`:带置信度的估计值(视觉识别结果、软计数),`confidence ∈ [0,1]`;
- `unknown`:不知道。**"未知"与"没有"是两种完全不同的信号**,协议与编码必须能区分(例:中途接入时换三张信息是 unknown,不是"没换")。

统计层的所有计算必须能在部分输入为 unknown/estimated 时**降级运行**:输出软值/区间/先验,而不是崩溃或给出错误的确定值。

编码器对 unknown 字段的处理:该区段填中性值 + 专门的"未知标志位"置 1(详见任务 7)。

### 第 1 层:事实层 facts(观测到什么记什么)

**A. 自己的私有信息**(部署时来自用户自己的手牌,可靠性最高,一般恒为 observed)

- 自己手牌:27 维计数向量(每种牌 0~4 张)
- 自己的定缺花色(万/条/筒,3 维 one-hot;未定缺阶段为空)
- 自己的副露:碰/明杠/暗杠/补杠(种类 + 牌面)
- 自己是否已胡
- 自己的过庄锁定状态:`passed_hu_lock` / `passed_fan`(放弃时番数基准,归一化)。部署时由用户端事实决定(用户自己知道有没有放弃过胡),可 unknown
- 自己换三张的换出/换入(**可 unknown**:中途接入时整体未知)

**B. 公开信息**(所有人可见;部署时来自视觉识别,可为 estimated)

- 各家公开副露:碰、明杠、补杠(他人暗杠:数量与亮明后的牌面属公开信息;未亮明的暗杠牌面不可见,不得编码)
- 各家弃牌:按时序的弃牌序列 + 各家弃牌计数(时序部分可截断:中途接入只有接入之后的段,接入前为 unknown)
- 各家定缺花色(公开;可 unknown——接入时未听到宣告且行为尚不足以确认)
- 各家是否已胡、已退出
- **已胡玩家亮出的完整手牌(v4 从推理层挪入,重要)**:血战到底胡牌要亮牌,那 13/14 张是确定事实,**整体计入 `seen_counts`**。尾盘多家已胡时,未知区急剧缩小,这是余牌精度的最大来源
- 当前庄家相对位置 / 自己是否庄(可 unknown)
- 牌墙剩余张数(可 estimated:由已观测事件推算;或 unknown)
- 是否海底/最后一张(可由牌墙估计导出,带不确定性)

**C. 牌局记忆**(玩家视角可记忆的时序信息;中途接入时只有接入后的部分)

- 换三张追踪:自己换出/换入的牌、方向、目标;换出的牌是否后来被公开(打出/碰/杠/胡牌亮牌)
- 事件历史:各家按巡数的弃牌序列、碰/杠/胡事件时间
- **观测起点标记 `observation_start`**:本次观测从第几手开始(第 0 手 = 全程观测;>0 = 中途接入),之前的历史一律 unknown

**D. 规则配置(v4 新增)**

- `rule_config`:封顶番 `max_fan`、自摸模式 `self_draw_mode`、底分。编码进观测,模型可条件化于不同玩法;部署时按目标牌桌配置填写

### 第 2 层:统计层 statistics(硬编码——有唯一正确答案的算术与逻辑)

由事实层**确定性计算**,全部可写验收测试:

- **精确/软计数**(`tile_counting.py` 核心):
  - 每种牌的已现张数 = 自己手牌 + 全部弃牌 + 全部公开副露 + 已亮明暗杠 + **已胡玩家亮出的手牌**
  - 观测为 estimated 时使用**软计数**:按置信度加权的期望张数,逐张截断到 [0,4],并按"全场共 108 张"守恒做全局一致性调和(见任务 5)
  - 每种牌的理论剩余张数 = 4 − 已现张数(软计数下为期望值),归一化
  - 各花色可见密度;未知区总量 = 108 − 已现总量
  - **未知区分解**:未知区 = 牌墙 + 各未胡对手的暗手牌,各对手手里几张是确定已知(或可由观测事件推算)的
- **约束先验**(仍属统计层——给定假设下的精确数学):
  - 超几何先验:P(某张在牌墙) = 牌墙余量 / 未知区总量,按各对手手牌张数分摊
  - **定缺硬约束**:定缺某门且长期未打该门的玩家,持有该门牌的概率随巡数趋近于零(有则不能胡,理应早打);尾盘近似硬约束
  - 已胡玩家不再摸牌,未知区动态收缩
- **自己牌理统计**(`hand_analysis.py`):向听数、有效进张集合与剩余张数(基于软计数)、对子/搭子/面子/孤张、潜在番型(平胡/对对胡/清一色/七对/龙七对/根)
- **候选动作统计**(`action_features.py`):每个候选弃牌后的向听/进张变化、是否破坏结构、是否打缺门、生张/熟张
- 当前轮到谁、当前可响应事件、抢补杠响应窗口

### 第 3 层:信念层 beliefs(学习式——无解析公式,靠训练)

**v4 明确:这一层的数字不再来自手写公式,来自训练出的 belief 网络。** S2 只负责定义接口、特征抽取、标签生成器与未训练时的先验回退(见任务 6)。

- 每种未知牌在牌墙 / 各对手手中的概率分布
- 各对手听牌概率、对各候选弃牌的点炮风险

监督信号:S1 引擎上帝视角提供每张牌的真实位置作为标签(**只用于训练时批改,严禁作为任何模型的输入特征**)。

**v4 明确移除**(不得出现在协议与编码中):断章分析、花色兴趣分数、攻防倾向、胡牌路线反推、`fragment_signals`、`late_game_belief_update`(其合理内核——"尾盘长期未见的牌信号更强"——由学习式 belief 自然习得,不手写)。

### 明确不编码的内容(沿 v3)

- 杠转水金额公式、流局退杠钱公式、一炮多响拆账、`next_dealer` 裁判结果——结算是引擎内部的事,模型只看可观测上下文(自己是否杠过、是否杠后打牌、是否海底、是否过庄锁定)。

## 三、任务清单

### 任务 1:统一协议 `protocol.py`(s2.v4)+ 协议文档

**做什么**:定义与来源无关的标准数据结构,新增可观测性维度。

- 每个事实字段用统一包装 `ObservedValue`:`{value, status: observed|estimated|unknown, confidence: float}`(observed 恒 1.0;unknown 时 value 为空)。容器类字段(弃牌序列等)支持"部分已知 + 起点之前 unknown"
- 三段结构:`facts` / `statistics` / `beliefs`(v3 的 `inferences` 更名并重定义:只装学习式 belief 的输出或先验回退,不装手写启发式)
- 基础字段:版本(`s2.v4`)、视角玩家、阶段、当前玩家、庄家、牌墙数量、玩家列表、合法动作、**`observation_start`**、**`rule_config`**
- S1 规则字段(沿 v3):`dealer_relative_position`、`is_dealer`、`is_last_tile`、`passed_hu_lock`/`passed_fan`(仅自己)、`public_gang_counts`、`own_concealed_kong_count`、`own_scored_kong_count`、`pending_added_kong`、`discard_passed`——**全部允许 unknown**
- 事实字段:`exchange_tracking`(可整体 unknown)、`event_history`、`revealed_win_hands`(已胡玩家亮牌,新增)、`seen_counts`(软计数,含期望值与不确定度)
- 统计字段:`remaining_tile_counts`、`unknown_pool_breakdown`(牌墙/各对手暗牌张数)、`own_hand_analysis`、`candidate_action_features`、`dingque_constraints`(各家定缺约束强度)
- 信念字段:`tile_location_beliefs`(每张未知牌在牌墙/各对手的概率)、`opponent_tenpai_beliefs`、`discard_danger`,并标记来源(`prior` 先验回退 / `learned` 模型输出)
- **移除字段**:`fragment_signals`、`late_game_belief_update`、`post_win_inference`(亮牌部分并入 `revealed_win_hands`/`seen_counts`,路线反推砍掉)、`opponent_beliefs` 中的花色兴趣/攻防倾向
- JSON round-trip 无损;`docs/protocol_spec.md` 同步字段语义、可观测性取值、示例 JSON(至少含:完整观测、中途接入、含识别噪声三个示例)

**验收**:协议能表达"完整决策点""中途接入决策点""带噪声决策点"三种情形;round-trip 无损;每个字段的可观测性语义在文档中无歧义。

### 任务 2:引擎适配器 `adapters/from_engine.py` + 可见性过滤 `visibility.py`

**做什么**:S1 `GameState` → 统一协议(训练数据主通路,产出**全 observed** 的完美观测)。

- `from_engine(engine_state, player_id)` → 该玩家视角协议对象,所有字段 status=observed、confidence=1.0、`observation_start=0`
- 玩家列表按相对位置排列:自己/下家/对家/上家
- 对手只隐藏暗手牌;手牌数量、公开副露、已亮明暗杠、弃牌、定缺、是否已胡照常可见
- **已胡玩家的亮牌进入 `revealed_win_hands` 并计入 `seen_counts`**(新增,须专项测试)
- `rule_config` 从引擎配置填充

**关键单测**(沿 v3 并扩展):
- 构造含自己暗杠、他人暗杠、已有一家胡牌的状态,对 4 个玩家分别验证:自己手牌可见、对手暗手牌不可见、他人暗杠牌面亮明后可见、**已胡玩家亮牌全部进入 `seen_counts`**
- 可见性无泄漏(红线,沿 v3 全部保留)

### 任务 3:合法动作判定 `legality.py` + 动作空间 `action_space.py`【架构调整】

**做什么**:把"哪些动作合法"做成**纯函数 `legal_actions(protocol_state) → 动作集合`**,训练与部署共用同一份代码。

背景:v3 规定 legal mask 只能来自 S1 引擎。训练时成立;但**部署时驱动牌局的是真实世界,现场没有引擎**,合法性必须从观测状态推出。这不是"分叉规则逻辑",而是把规则判定从引擎中提出来共享——引擎内部也应改为(或验证等价于)调用同一函数。

- 动作空间沿 v3 不变:打 27 种牌 / 碰 / 明杠 / 补杠 / 暗杠 / 胡 / 抢杠胡 / 过 / 换三张选择 / 定缺 3 选 1
- `legality.py` 实现:定缺约束、过庄锁定(含更大番解锁)、海底禁杠、抢杠胡响应窗口、阶段约束(换三张/定缺/对局)
- **残缺观测下的行为**:合法性主要取决于自己手牌(observed)与最近弃牌(observed/estimated),大多可精确判定;当某前提字段为 unknown 时(如过庄锁定状态未知、牌墙是否最后一张未知),按**宽松策略**给出动作并在协议中标记 `conditionally_legal` 及其依赖假设——辅助模型给建议时可附带前提,最终由人执行
- `action_space.py`:动作 ↔ 索引双向映射;`legal_mask(protocol_state)` 调用 `legality.py`

**验收**:
- **训练态交叉验证**:用 S1 引擎跑 ≥1000 局,每个决策点比对 `legality.py` 输出与引擎 `legal_actions`,**零不一致**;特别覆盖海底禁杠、过庄锁定/解锁、一炮多响可胡、抢杠胡窗口
- 残缺态:构造"过庄状态 unknown""牌墙数 unknown"场景,验证降级行为与 `conditionally_legal` 标记正确
- 动作 ↔ 索引双向无误

### 任务 4:观测退化 `observation_degradation.py`【v4 核心新增】

**做什么**:把 from_engine 产出的完美观测,退化成与部署分布一致的残缺观测。这是弥合 sim2real gap 的关键;**S5 必须在退化观测上训练**(写入 S5 前置约束)。

实现为可组合、可配置概率的退化算子,作用于协议对象(from_engine 之后、编码之前):

1. `MidGameSnapshot(k)`:模拟中途接入——截断第 k 手之前的全部历史(弃牌时序、事件历史、换三张、巡数),`observation_start=k`,被截断信息置 unknown;但**当前可见的静态牌面保留**(牌池里躺着的牌、公开副露——中途接入也看得见),只是失去时序与来源
2. `MaskExchange(p)`:以概率 p 把换三张追踪整体置 unknown
3. `VisionNoise(confusion_matrix, miss_rate)`:按视觉模型的混淆矩阵对弃牌/副露牌面做替换扰动、按漏检率删除观测,并把受扰字段 status 降为 estimated、填入相应置信度。混淆矩阵先用合理占位(邻近点数/同点数异花色易混),视觉线有实测数据后替换
4. `MaskField(field, p)`:随机把次要字段(巡数、牌墙计数、庄家、各家定缺)置 unknown
5. 组合器:按"部署难度画像"配置一组算子与概率,支持课程式(由易到难)调度;同一局引擎数据可退化出多份训练样本

**硬约束**:
- 退化只作用于观测,**不改变引擎真实状态与标签**(上帝视角标签仍然精确)
- 退化后仍是合法协议对象,统计层/编码器/合法性判定全链路可运行
- 同随机种子同输出(可复现)

**验收**:每个算子有专项单测;退化后协议 schema 校验通过;`MidGameSnapshot` 后统计层仍能给出软计数(静态牌面仍计入);全链路(退化 → 统计 → 编码 → legal mask)在 1000 个退化样本上不报错。

### 任务 5:确定性计数 `tile_counting.py`【硬编码,有 oracle】

**做什么**:第 2 层统计中与"数牌"有关的全部确定性计算。

- 精确计数(全 observed 时):已现张数、理论剩余、花色密度、未知区总量与分解(牌墙 vs 各对手暗牌张数)
- **软计数**(存在 estimated 观测时):置信度加权期望张数;逐张截断 [0,4];**全局守恒调和**——所有牌的(已现期望 + 未知区)总和必须恒等于 108,出现矛盾(如某张期望已现 >4)时按置信度最低的观测回调,并输出矛盾标志(供上层提示"识别可能出错")
- 约束先验:超几何分布(每张未知牌在牌墙/各对手的先验概率);定缺约束强度(定缺某门、连续 n 巡未打该门 → 持有概率衰减曲线,曲线形状是确定性规则而非拍脑袋的行为猜测:其依据是"有缺门牌不能胡,理性玩家必然尽早打光")
- 已胡玩家亮牌、退出摸牌对未知区的更新

**验收(oracle 对账,重点)**:
- 全 observed 态:与 S1 上帝视角逐张对账,**已现/剩余/未知区分解必须 100% 精确相等**(蒙特卡洛 ≥1000 局全过)
- 软计数态:注入已知噪声,验证期望计数无偏、截断与守恒调和正确、矛盾检测触发
- 超几何先验:与暴力枚举/蒙特卡洛模拟对拍,误差在统计容差内
- unknown 输入降级:牌墙数 unknown 时先验退化为均匀假设而非报错

### 任务 6:学习式信念 `tile_belief.py`【接口 + 先验回退 + 标签生成器】

**做什么**:定义信念层的生产方式。**本任务不训练模型**(训练在 S4/S5 时间线),但把接口、特征、标签、回退全部备好,保证 S2 交付时全链路可运行。

1. **统一接口** `TileBelief`:输入协议状态(facts+statistics),输出 `tile_location_beliefs`(27 种牌 × {牌墙, 下家, 对家, 上家} 的概率,满足行归一与守恒约束)、`opponent_tenpai_beliefs`、`discard_danger`
2. **先验回退实现 `PriorBelief`**(确定性,可测):直接采用 `tile_counting` 的超几何先验 × 定缺约束,不含任何行为学猜测。这是 belief 网络训练完成前的默认实现,也是它必须击败的基线
3. **学习实现占位 `LearnedBelief`**:加载训练好的网络,推理输出;S2 阶段只定义接口与输入特征规范
4. **标签生成器**(S2 交付物):从 S1 上帝视角生成监督标签——每个决策点、每张未知牌的真实位置(牌墙/哪个对手)、各对手是否听牌、每张候选弃牌是否真实点炮张。与观测退化组合:**输入是退化观测,标签是精确真值**——这正是让 belief 网络学会"从残缺信息推断"的训练配方
5. **红线**:上帝视角只进标签,任何情况下不得进入模型输入特征;沿用 v3 的"同可见信息同输出"测试思想——两个隐藏牌不同但可见信息相同的状态,输入特征必须相同(标签可以不同,这正是不确定性的来源)

**验收**:接口三实现可互换;`PriorBelief` 输出满足归一/守恒并与任务 5 对拍;标签生成器在 100 局引擎数据上产出格式正确、与上帝视角逐张一致的标签;可见性红线单测通过。
(belief 网络本体的训练与"必须显著优于 `PriorBelief`、校准误差达标"的验收,列入 S4 任务清单。)

### 任务 7:状态编码器 `encoder.py`【未知感知】

**做什么**:(可能残缺的)统一协议 → 定长张量。

- 形状固定、确定性、数值归一化、相对位置编码(沿 v3)
- **未知感知编码(新增)**:每个可 unknown 的区段配一个"未知标志位";unknown 时数值区填中性值(0 或先验)且标志位=1;estimated 时附置信度维度。模型由此能区分"没有 / 不知道 / 大概是"
- 编码区段:自己手牌与牌理、各家公开信息、`seen_counts`(软计数+不确定度)、`remaining_tile_counts`、未知区分解、`tile_location_beliefs`(来自 `TileBelief` 当前实现)、庄家/过庄/杠上下文/海底、阶段与响应窗口、`observation_start` 与观测完整度摘要、`rule_config`、候选动作特征
- 打印/文档化编码结构表(每区段 offset 与含义)

**验收**:同输入同输出;关键字段可还原;**对完美观测与三种退化观测(中途接入/遮换三张/噪声)都能编码且形状一致**;unknown 标志位行为正确;无可见性泄漏。

### 任务 8:视觉适配器 `adapters/from_vision.py`【从"预留"升级为正式设计】

**做什么**:定稿视觉线 → 协议的事件流 schema 与调和逻辑。视觉识别模型本体不在 S2 范围,但 schema 必须现在定,否则协议日后必然破坏性升版。

- **事件 schema** `VisionEvent`:`{seq, timestamp, event_type(discard/pong/kong/hu/deal/dingque…), player(相对位), tile, confidence, alternatives[(tile, conf)…]}`;另有**快照事件**(接入时对牌池/副露的一次性静态识别)
- 适配器职责:事件流(可从任意时刻开始)→ 累积成协议状态;置信度传入 `ObservedValue`;`observation_start` 按首个事件确定
- **一致性调和**:利用硬约束(每种牌至多 4 张、全场 108 张、各家手牌数与摸打事件守恒)检测前后帧矛盾,矛盾时降置信度/回滚该观测并输出矛盾报告
- 与任务 4 的对应关系:观测退化模拟的正是本适配器的输入分布,两者的噪声模型参数应同源(同一份混淆矩阵配置)

**验收**:手工构造的事件流(含中途接入、含错误识别、含矛盾)能正确转成协议;矛盾检测触发并可审计;与 from_engine 产出的协议通过同一 schema 校验。

### 任务 9:牌局记忆 `memory.py`

沿 v3 缩减:换三张追踪(可整体 unknown)、事件历史摘要、观测起点管理。移除"疑似保留巡数""长期保留花色信号"等行为学推断(它们属于被砍掉的手写启发式)。**新增**:所有记忆结构必须支持从任意 `observation_start` 开始为空、随观测增量生长。

**验收**:中途接入时记忆结构合法且不虚构接入前信息;换三张追踪在 observed 态行为与 v3 一致。

### 任务 10:端到端贯通 + 回归验证 + 知识注解

1. **完美观测链路**:S1 引擎跑 ≥100 局,每个决策点:from_engine → 统计 → belief(PriorBelief)→ 编码 → legal mask,全程不报错、形状一致、可见性无泄漏
2. **退化观测链路(新增)**:同样 100 局,叠加观测退化后全链路运行;抽样验证软计数守恒、unknown 标志位、`conditionally_legal` 标记
3. **视觉模拟链路(新增)**:用退化配置生成模拟视觉事件流,走 from_vision → 全链路
4. **legal mask 交叉验证**:≥1000 局,`legality.py` vs 引擎零不一致(完美观测态)
5. **同可见信息同输出**:隐藏牌不同、可见信息相同的两状态,facts/statistics/beliefs 输入特征/编码全部相同
6. **标签管线**:标签生成器 + 退化观测配对样本产出 ≥10000 个决策点的训练样本文件,格式校验通过
7. `docs/concepts.md` 注解(供学习):什么是可观测性建模与 sim2real gap;为什么"未知"要显式编码;为什么计数硬编码而信念要学习(oracle 判据);域随机化/观测退化的原理;belief 标签为什么可以用上帝视角而输入不可以;动作掩码、相对位置编码、定长张量(沿 v3)

**验收**:三条链路全通;交叉验证零不一致;学习文档能讲清"残缺信息下的决策输入层"这一定位。

## 四、全局技术约束

- Python 3.10+;numpy 为主;S1 引擎作为依赖 import(仅训练态;**部署态代码路径不得 import 引擎的对局推进逻辑**,只允许共享 `legality.py` 与纯规则计算)
- **可见性是红线**(沿 v3):任何"玩家视角"产物不得含他人暗手牌;上帝视角只允许出现在两处——调试工具、belief 标签生成器
- **可观测性是新红线**:任何统计/编码代码不得把 unknown 当 0 或当"没有"处理;新增字段必须声明可否 unknown 及降级行为
- **同可见信息同输出**(沿 v3):可见信息相同 → facts/statistics/belief 输入/编码全部相同
- **硬编码/学习的分界必须守住**:统计层禁止出现无 oracle 的行为学参数;信念层禁止手写公式(先验回退除外,它是可验证的数学)
- 编码确定性、可序列化、批量化;协议版本 `s2.v4`,后续演进显式升版
- 观测退化的噪声参数集中配置(与 from_vision 共享),可随视觉线实测数据更新
- 每模块 pytest 单测;可见性、可观测性降级、计数 oracle 对账、legal mask 一致性是重点测试对象
- 不把杠转水、流局退杠钱等结算公式编码进模型输入(沿 v3)
