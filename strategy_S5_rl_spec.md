# S5 需求规格:自博弈强化学习(从"会打牌"到"打得强")

> 策略 AI 项目第五阶段。S1 引擎(环境+裁判)、S2 状态层(s2.v4)、S3 规则基线(对手池三档)、S4(belief 网络 + 模仿策略 + 竞技场)均已交付。
> 本阶段用自博弈强化学习,把 S4 的"S3 水平模仿策略"提升为**显著超越 S3 的强策略**——真正的牌力从这里练出来。
> 交给 AI 编码助手执行。Python 3.10+,PyTorch,复用 S4 训练基础设施。

---

## 设计定位(给执行者的关键认知)

**S5 是整条线里第一个"没有老师"的阶段。** S3 是手写规则、S4 是模仿老师,上限都被人的水平锁死;S5 靠自己跟自己打、用输赢分数当唯一裁判,水平上限只受算力和训练方法限制。前面刻意不在 S3/S2 里堆高级战术(防守预判、弃和搏大、形势判断),就是留给这里学的。

**三条前置约束(在 S2 v4 与 S4 文末已锁定,本阶段不可讨价还价)**:

1. **训练观测必须经过观测退化**(课程式从轻到重)。最终产品是现实世界辅助模型,输入残缺有噪声;只在完美观测上练出的强者是"温室强者",部署即崩;
2. **对手池多样化**。只跟最新的自己打,会练出"只会克制自己"的偏科策略,遇到风格不同的真人就失灵。对手池 = 随机/贪心/S3(S3 已交付)+ 自己的历史版本快照;
3. **评估必须含残缺观测赛道**,不得只看完美观测胜率。

**S5 与 S4 的资产关系**:策略网络从 S4 模仿权重初始化(不从随机开始);belief 网络冻结复用(读牌能力已单独验收过);arena、数据管线、退化算子、metrics 全部复用。S5 新写的只有:自博弈采样、PPO 训练循环、对手池管理、奖励与课程调度。

## 一、核心设计决策(定死,避免执行时发散)

### 1. 算法:PPO(近端策略优化)

- 选 PPO 的理由:对超参相对稳健、天然支持动作掩码(非法动作 logit 置 −inf)、单机可跑、社区成熟。**不要上更花哨的算法**(MCTS 混合、CFR 类、模型基 RL),先把 PPO 跑到收益平台期,算法升级是后续优化项且必须以对比数字为理由;
- 策略头 + 价值头:策略头结构与 S4 策略网络一致(从其权重初始化);价值头新增,预测"本局从当前决策点起的最终得分期望"(从零训练);
- **早期 KL 约束**:训练初期对 S4 模仿策略加 KL 散度正则(防止一开始就把冷启动权重冲垮、退回乱打),随训练进程衰减到 0。

### 2. 奖励:只用最终得分,不做人工塑形

- 每局结束,S1 结算给出的**该玩家最终净得分**(含胡牌分、杠钱、转水、查叫,零和)就是全部奖励,按底分与封顶归一化;
- **禁止人工中间奖励**(如"向听下降+0.1""碰牌+0.05"):中间奖励全是人的偏见,会被策略投机(reward hacking)——这正是 S2 砍掉手写启发式的同一条原理在奖励侧的翻版。信用分配难的问题交给价值网络与足够的对局量解决;
- 血战到底特性:玩家胡牌后本局仍在继续(杠钱转水、查叫都可能再变动分数),**episode 以"本局完全结束、结算完成"为界**,胡牌不是 episode 终点。

### 3. 自博弈与对手池 `league`

- 每局 4 个座位,**1 个学习者 + 3 个从对手池采样的对手**(同桌可混搭);
- 对手池构成与采样权重(可配,给默认值):最新自己 35% / 历史快照 35%(均匀采历史)/ S3 20% / 贪心+随机 10%;
- 每 N 步(可配)把当前策略快照入池;池子容量上限,淘汰最老的(保留几个里程碑版本永不淘汰);
- **反遗忘门槛**:新快照入池前,必须在 arena 里对池内全部成员胜率不低于阈值(防止"新版本克制旧版本 A 却输给更旧的 B"的循环退化)。

### 4. 观测退化课程 `curriculum`

- 学习者座位的观测按课程退化:训练前期以完美/轻噪声为主,逐步提高中途接入、重残缺画像比例;最终混合比例对齐部署预期(与 S4 数据管线用同一套画像配置);
- 对手座位默认完美观测(对手是"正常打牌的人",不残缺的是他们的眼睛,残缺的是学习者的眼睛——与部署情形一致);
- 课程推进条件写成显式规则(如"当前难度下对 S3 胜率达标后升难"),不靠拍脑袋换档。

### 5. belief 网络:先冻结,后可选微调

- 第一阶段冻结 S4 的 belief 网络(它有独立真值验收,不要让 RL 梯度污染它);
- 策略到达平台期后,**可选**开放 belief 联合微调作为对比实验——但微调后的 belief 必须重跑 S4 任务 2 的全部验收(优于先验、校准达标),不达标就回滚冻结。

## 三、任务清单

### 任务 1:自博弈采样 `rollout.py` + 奖励 `reward.py`

- 用 S1 引擎并行跑局(多进程/向量化,可配并行度);学习者座位走"退化观测 → 编码(belief 冻结填充)→ 策略采样动作"链路,对手座位走各自策略;
- 收集学习者视角轨迹:(编码状态, 动作, legal mask, log-prob, 奖励, done);奖励只在局末发放(结算净得分归一化),中间步奖励恒 0;
- **验收**:并行跑 ≥1000 局无崩、零非法动作、零和守恒;轨迹格式经 schema 校验;吞吐给出数字(局/分钟——这决定训练可行性,若过慢先做性能优化再进任务 2);同种子可复现。

### 任务 2:PPO 训练循环 `ppo_trainer.py` + `train_rl.py`

- GAE 优势估计、PPO clip、熵正则、价值损失、对 S4 策略的 KL 正则(可衰减);梯度裁剪;
- 断点续训(checkpoint 含优化器状态、league 状态、课程进度);训练曲线落盘(得分、熵、KL、价值损失、对 S3 胜率);
- **训练健康监控(必须实现,RL 最容易静默烂掉)**:熵骤降(策略过早僵化)、KL 爆炸(更新过猛)、价值损失发散、对 S3 胜率回退——任一触发时自动告警并保留现场 checkpoint;
- **验收**:小规模冒烟(短训练)后对 S3 胜率曲线呈上升趋势;健康监控各项能被构造场景触发;断点续训前后曲线连续。

### 任务 3:对手池 `league.py`

- 快照入池/淘汰/里程碑保留;采样权重可配;反遗忘门槛(入池前 arena 过关);
- **验收**:构造"新策略克制 A 输给 B"的模拟场景,验证门槛能拦截;池状态可序列化、续训后一致;对局中同桌混搭对手正确执行各自策略。

### 任务 4:观测退化课程 `curriculum.py`

- 画像比例调度(配置驱动);升难条件显式化;当前难度写入训练日志与 checkpoint;
- **验收**:课程按配置推进、可复现;学习者与对手的观测退化配置隔离正确(对手恒完美观测);抽样统计画像比例与配置一致。

### 任务 5:阶段评估与最终验收(用 S4 的 arena)

训练分里程碑评估,**每个里程碑都跑双赛道**:

- **完美观测赛道**:1 个 S5 策略 + 3 个 S3,≥10000 局,平均得分与胜率;
- **残缺观测赛道**:S5 座位施加部署画像退化(含中途接入),对手完美观测,≥10000 局;
- 对照组:S4 模仿策略在同条件下的成绩(证明 RL 确实带来提升,而非评估口径变化);
- 附带检查:价值头预测与实际得分的相关性(价值网络是否学到东西)、策略熵分布(是否僵化)、各番型/胡牌率分布(风格画像)。

**最终验收(全部给出数字)**:
1. 完美赛道:对 S3 平均得分**显著为正**且相对 S4 模仿策略有明显提升(显著性用置信区间说话);
2. 残缺赛道:对 S3 平均得分仍显著为正(允许低于完美赛道,幅度写进报告)——**这是产品可用性的判据,比完美赛道更重要**;
3. 反遗忘:最终策略对池内全部历史版本与三档基线胜率不低于门槛;
4. 零非法动作、零和守恒(全程)。

### 任务 6:高手行为验证(人机协同验收)

- 用此前整理的**"8 条高手直觉"验证清单**逐条构造/筛选典型局面(如:尾盘该弃和时是否收手、危险张是否会扣、定缺压力下的取舍、余牌明显偏紧时的路线切换等,以清单原文为准);
- 对每条:从自博弈对局中自动筛出符合条件的决策点 + 构造针对性测试局面,输出 S5 策略的选择与 S3/S4 的选择对比,形成人工评审材料(局面图 + 三方选择 + belief 输出);
- **验收**:8 条清单每条都有 ≥10 个样本局面的对比材料;由我(人类)逐条评审打分,多数条目达到"像高手"判定。此项不设自动化硬门槛——高手直觉本就是人来验的。

### 任务 7:知识注解 `docs/concepts.md`

- 自博弈为什么能超越老师;PPO/优势函数/价值网络的直白解释;
- 为什么奖励只用最终得分(reward hacking 风险);为什么要对手池(自博弈退化/循环克制);为什么退化课程从轻到重;
- 不完美信息四人博弈没有理论最优保证,工程上如何用多样对手 + 双赛道评估逼近实用最优;
- **验收**:学习者读完能说清 S5 的水平从哪来、三条前置约束各防什么坑。

## 四、全局技术约束

- Python 3.10+;PyTorch;复用 S4 基础设施,不重复造轮子;
- **可见性红线**(全程继承):学习者输入永不含隐藏信息;belief 标签路径在 S5 不使用(除非开启微调对比实验,且须重跑 S4 验收);
- **退化训练是默认**:完美观测专属训练只能作为对照实验;
- 全流程断点续训、同种子可复现;checkpoint 带 league 与课程状态;
- 训练吞吐是一等公民:rollout 性能不达标先优化再扩训练(S1 单局 <2ms 的目标在此兑现价值);
- 每模块 pytest 单测;league 门槛、课程隔离、轨迹零泄漏、健康监控是重点测试对象。
