## 一、规则全集(引擎必须精确实现,这是与执行者的规则契约)

### 1. 牌池与基本结构
- 只有万(W)、条(T)、筒(B)三门,每门 1~9,各 4 张,共 108 张。无字牌、无花牌。
- 4 名玩家。
- 胡牌牌型:**标准型 = 4 个面子 + 1 对将**;或 **七对 = 7 个对子**。面子 = 刻子(三张相同)或顺子(同门连续三张)。

### 2. 流程
- 发牌:每人 13 张,庄家 14 张(先打一张)。
- **换三张**(发牌后、定缺前的开局环节,详见 1.5):掷两骰定方向,每人选 3 张**同一花色**的牌,三家**同步暗换**。
- **定缺**:换三张之后,每人指定一门花色为"缺门"。**必须打光手中所有缺门牌后才允许听牌/胡牌**;手里还有缺门牌时不能胡。
- 开局顺序固定为:**发牌 → 换三张 → 定缺 → 庄家首打 → 进入摸打循环**。

### 1.5 换三张(开局特殊规则,必须精确实现)
- 时机:发牌后、定缺前。
- **方向由两颗骰子点数之和决定**,全桌统一方向(对家=对面玩家,下家=顺时针下一位,上家=逆时针上一位):

  | 骰子总和 | 方向 | | 骰子总和 | 方向 |
  |---|---|---|---|---|
  | 2 | 下家 | | 8 | 上家 |
  | 3 | 对家 | | 9 | 对家 |
  | 4 | 上家 | | 10 | 下家 |
  | 5 | 对家 | | 11 | 对家 |
  | 6 | 下家 | | 12 | 上家 |
  | 7 | 对家 | | | |

  规律:以 2 起,按【下家、对家、上家、对家】四步循环。**对家占所有奇数点(3/5/7/9/11),概率最高(1/2);下家(2/6/10)、上家(4/8/12)各约 1/4。**
- **硬性约束:每人必须拿出正好 3 张同一花色的牌**(三张万 / 三张条 / 三张筒,不可混花色)。引擎强制此约束;若玩家选择不合法(不足3张同花色或混花色)则为非法动作。
- **同步暗换**:三人各自暗选 3 张,同时按方向传给目标玩家,过程互不可见(信息不对称,属不完美信息的一部分,可见性分离需覆盖)。
- 设计说明:**"换哪三张"是策略决策点**(送牌泄露信息、换回同花色暗示牌型差等技巧),交由 AI(S5)学习;引擎只强制"同花色3张"这条硬规则。因此换三张必须作为**开局动作**进入动作空间。
- 轮转:摸牌 → 打牌,其间可碰/杠/胡(按麻将标准优先级:胡 > 碰/杠 > 摸牌)。
- **血战到底**:某家胡牌后,该家退出后续摸打(已胡),**牌局继续**,其余玩家接着打,直到**未胡家数 ≤ 1**(其余都已胡,一炮多响可一步从剩 2 家跳到剩 0 家),或牌墙摸完。一局中可有多家先后胡牌;一张牌可被多家同时点炮胡(一炮多响,见 4.6)。
- **庄家轮转**(见 4.7):本局有人胡 → 第一个胡牌者下局当庄(首胡即一炮多响则放炮者当庄);全程无人胡的荒庄 → 当局庄家连庄。

### 3. 番型与算番(算番阶段;可累加;封顶只封番)
基准:**底分可配置(默认 1 分)**;番→分:**底番=2^0=1分,n 番 = 2^n 分**(乘以底分)。
- **平胡** = 底番(0 番加成)
- **对对胡**(全刻子+将)= +1 番
- **金钩钓**(手里只剩 1 张钓将,其余 4 个面子全是碰/杠副露,单钓将)= **2 番**(此番**替代对对胡**,出现金钩钓时不再单独计对对胡;金钩钓本身就是对对胡的极致形态)
- **清一色**(全一门)= +2 番
- **七对** = +2 番
- **根**:任意 4 张完全相同的牌(无论是否杠出、是否在七对中),**每根 +1 番**
  - **杠没有独立的"杠番"**:杠的收益只有两处——① 杠出来的 4 张构成一个根(+1 番)② 刮风下雨的即时杠钱(直明杠/暗杠,见第四节,独立结算)。算番时**不再为"杠"这个动作额外加番**(杠上花/杠上炮/抢杠胡是"彩头番",不属于杠番,照计)。
- **杠上花/杠上炮** = +1 番:
  - 杠上花:杠后补摸的牌直接自摸胡 → 胡牌者 +1 番
  - 杠上炮:杠后打出的牌被他人点炮胡 → 胡牌者 +1 番
- **抢杠胡** = +1 番:他人补杠(碰→杠)的那张牌正好是自己的胡牌张,抢在补杠成立前胡牌 → 胡牌者 +1 番(暗杠不可被抢)
- **海底** = +1 番(围绕牌墙最后一张牌胡牌):
  - 海底捞月:摸到牌墙最后一张牌并自摸胡 → +1 番
  - 海底炮:摸到最后一张牌者打出后被他人点炮胡 → 胡牌者 +1 番
- 番型自然累加,不特判组合:
  - 龙七对 = 七对(2)+ 1 根(1)= 3 番;2 根 = 4 番
  - 清对(清一色+对对胡)= 2+1 = 3 番
  - 清七对(清一色+七对)= 2+2 = 4 番
  - 金钩钓 = 2 番;金钩钓+1根 = 3 番;清一色金钩钓 = 2+2 = 4 番
  - 金钩钓 + 杠上花/杠上炮/抢杠胡 = 2+1 = 3 番(牌型番与方式/彩头番独立叠加)
  - 金钩钓 + 自摸 = 2 番后再按自摸配置(加底/加番)处理
- **番型的替代/互斥关系(算番逻辑必须正确处理)**:
  - **金钩钓替代对对胡**:满足金钩钓时计 2 番,不再额外计对对胡 1 番
  - **七对与对对胡/金钩钓互斥**:七对是 7 个对子结构,不可能同时是对对胡(4面子+将);算番时按实际牌型结构二选一
  - **清一色、根、方式番(自摸/杠上花炮/抢杠胡)与上述牌型番独立叠加**
  - 算番总流程:先确定基础牌型番(平胡 / 对对胡 / 金钩钓 / 七对,四选一,金钩钓优先于对对胡)→ 叠加清一色 → 叠加每根 → 叠加方式/彩头番(杠上花炮、抢杠胡、海底)→ 封顶 → 应用自摸模式
- **封顶**:配置项 `max_fan`(默认建议 4),最终番数 = min(累加番, max_fan);**只封番,杠的即时分不受封顶影响**。
- **自摸**:配置项 `self_draw_mode`,二选一(下面算的是**每个赔付家各自支付的分数**;自摸时所有未胡家各付一份全额):
  - `add_di`(加底):每家应付 = 番对应分 + 1 个底分。例:七对(4分)自摸 = 4+1 = 5 分,其余每家各付 5
  - `add_fan`(加番):番数 +1 再算分。例:七对(2番)自摸 = 3番 = 8 分,其余每家各付 8
- **天胡/地胡/十八罗汉等极端番型**:本版不实现,预留配置开关位(默认关)。

### 4. 杠的即时结算(刮风下雨;独立于番,立即结算,不受封顶影响)
以底分为基准(默认 1 分,变则等比缩放):
- **直明杠**(杠别人打出的牌):被杠者付 2 分给杠牌者
- **暗杠**(手中 4 张):其余每家各付 2 分给杠牌者
- **补杠/旋风杠**(碰后摸到第 4 张补杠):**不结算任何杠钱**——补杠只改变牌形(把碰升级为杠、多摸一张、构成一个根),没有刮风下雨的钱;但仍可被抢杠胡
> 这是杠发生时立即结算的分,与最终胡牌算番完全独立、两套账。

**4.1 补杠没有杠钱(重要)**
- **补杠(碰→杠)一律不结算杠钱**:无论摸到第 4 张当回合立即补,还是留几手后再补,都只改变牌形(成根 + 多摸一张 + 博杠上花/承担被抢杠胡风险),**没有任何刮风下雨的钱**;也可以选择始终不补。
- 因此引擎**无需** `missed_instant_gang` 之类"即杠/迟杠"标记:补杠动作始终合法,结算时一律不产生杠钱。
- **暗杠**:手中 4 张,任何时候杠都正常结算(每家 2 分)。
- **直明杠**:即时发生即时结算(被杠者付 2 分)。

**4.2 抢杠胡(补杠可被抢,暗杠不可)**
- 当某玩家发起**补杠**时,该补杠的牌若正好是其他玩家的胡牌张,其他玩家可**抢杠胡**——抢在补杠成立前胡牌。
- 抢杠胡的胡牌者 **+1 番**(见番型表)。
- **暗杠不可被抢**;直明杠不涉及(杠的是别人刚打出的牌)。
- 引擎动作判定顺序:玩家发起补杠 → 引擎**先检查其他玩家是否可抢杠胡**(其 legal_actions 出现"抢杠胡"选项)→ 有人抢则**补杠作废、转为该玩家胡牌(+1番)并按胡牌流程结算**;无人抢则补杠成立(补杠本身无杠钱,见 4.1)。
- **多家可同时抢杠胡**:同一张补杠牌若多家都能胡,按一炮多响处理(各自独立结算,放炮者即补杠者)。
- 策略含义:摸到第 4 张"补杠(做成根、多摸一张博杠上花,但暴露信息且可能被抢杠胡)vs 不杠(保灵活、避抢杠胡风险)"是一个决策点,交由 AI 学习。注意补杠没有杠钱,这里权衡的是**牌形/摸牌机会 vs 信息暴露与被抢风险**,不是钱。

**4.3 杠转水(杠后点炮,杠钱转给胡牌者)**
- 触发:玩家杠牌(已即时收取杠钱)后,**紧接着打出的那一张牌**点炮被他人胡。
- 处理:该杠的**全部杠钱,全额转给胡牌者**(杠牌者吐出这笔已收的杠钱,转付给胡牌者)。
- 与杠上炮配套:此时胡牌者**同时**获得 ① 杠上炮 +1 番(见番型表)② 该杠钱转水。两者一起发生。
- 数额示例(底分 1,暗杠):暗杠时 3 家各付杠者 2 分 → 杠者 +6、三家各 -2。杠后打出的牌点炮被其中一家(记作 H)胡 → 这 6 分全额转给 H:杠者 -6、H +6。
  - 只看杠钱这一笔流水的净额:杠者 0(+6 收、-6 转出);另两家各 -2;胡牌者 H 净 **+4**(它作为暗杠付款人之一先付了 -2,转水又收回 +6)。四家合计 -2-2+0+4 = 0,零和成立。(H 的点炮胡牌分另算,这里只算转水这一笔。)
  - 实现口径:转水 = 在分数账上**追加**一条"杠者 → 胡牌者"的流水,把这笔杠钱原路从杠者账上扣除、加到胡牌者账上;**不删除原始杠钱记录,只追加转水记录**,便于回放与流局审计。
- 范围:**仅"杠后紧接打出的那一张点炮"触发**;杠后打安全牌未点炮、或过了若干轮后才点炮,**不转水**。自摸、正常结束不触发(除 4.4 荒庄特例)。
- **转水只适用有杠钱的杠(直明杠 / 暗杠)**;**补杠没有杠钱,故补杠后点炮不发生转水**(但补杠后打出点炮仍计杠上炮 +1 番——杠上炮是番,不是钱)。
- 引擎需记录每个杠:杠者、类型、收取金额、向谁收取、第几手杠,以便点炮时精确转走对应杠钱。

**4.4 荒庄时未听牌者杠钱失效(转水以"杠生效"为前提)**
- **仅"牌墙摸完的荒庄"触发**(因 ≥3 家胡而提前结束的牌局不触发,杠钱全保留,见第 6 节)。
- 荒庄时:**未听牌(未停牌)的玩家,本局作为杠者收取的所有杠钱全部失效**——**原路退回给当初支付杠钱的人**。
- **转水以"杠生效"为前提**:若某个未听牌者的杠在牌局中途已"杠后点炮转水"给了某胡牌者,该杠现在判失效 → **这笔转水也随之作废**:从受让的那个胡牌者账上扣回转水金额,连同杠钱一起原路退回原付款人。(胡牌者的正常点炮胡牌分不受影响,只回退那笔转水。)一句话:杠钱没了,转水也就不存在了。
- **已听牌者收取的杠钱正常保留**,不受影响。
- 这与查大叫是配套惩罚:荒庄时"听牌与否"决定杠钱保留与否(听牌者保、未听牌者退)。
- 实现:荒庄结算阶段,遍历每个杠,若杠者未听牌则按流水记录原路退款(含回退其已发生的转水);务必保证退款后总分仍零和守恒。

**4.5 杠的其他细则**
- **海底不能杠**:牌墙只剩最后一张时,不能杠(杠需补摸,无牌可补)。`legal_actions` 在此时不返回任何杠动作。杠的补牌与正常摸牌**消耗同一个牌墙计数**(本引擎不单设"杠尾"),"海底/最后一张"按这个统一计数判定。
- **连续杠(杠上杠)**:杠后补摸又能杠,可连续杠。每次杠的杠钱**各自即时结算**(连杠2次收2次钱)。
- **连续杠的杠上炮与转水**:以**最后一个杠之后打出的那张牌**为准——若它点炮,胡牌者得杠上炮+1番;转水**只退最后那个杠的钱**(前面各次杠的钱不退)。

### 4.6 一炮多响(一张牌同时多家可胡)
- 某人打出一张牌,若**同时有多家**能用它点炮胡,**全部可胡**(血战到底常见)。
- 放炮者**分别**赔付给每个胡牌者(各自按各自番型独立结算)。
- **杠后点炮且一炮多响时**:
  - 杠上炮 +1 番:**每个胡牌者都加**。
  - 转水杠钱:**对每个胡牌者各给一份完整转水钱(复制式,非平分)**。例:明杠杠钱 2 分,2 家多响 → 给每个胡牌者各转 2 分,杠者共吐 4 分;胡的人越多杠者吐越多。
- 海底炮一炮多响同理:每个胡牌者各得海底 +1 番。

### 4.7 庄家轮转(决定下一局庄家)
- **本局有人胡**:本局**第一个胡牌**的玩家,下一局当庄。无论本局是"≥3 家胡提前结束"还是"牌墙摸完的荒庄但中途有人胡过",只要有人胡过,都由首胡者当庄。
- **首胡即一炮多响**(本局第一次胡就是多家同响):**放炮的那个玩家**下一局当庄(而非胡牌者)。
- **无人胡的荒庄**(牌墙摸完且全程 0 家胡):**当局庄家连庄**(继续当庄)。
- 引擎每局结束按此规则确定下一局庄家,需追踪"第一个胡者 / 是否首胡即多响 / 本局是否有人胡过"。

### 4.8 其他对局规则
- **报听后可改听**:听牌后续摸打仍可改变听牌方向,不锁定(引擎无需特殊处理,正常自由出牌)。
- **不考虑诈胡**:不实现诈胡惩罚;`win_check` 仅在真正可胡时返回可胡,故不会出现诈胡。



### 5. 过庄(限制贪胡/养牌)
- 触发:玩家**已可胡他人的牌但选择不胡**(放弃一次"点炮胡"或"抢杠胡"的机会;放弃自摸不算,自摸本就不受过庄限制)。〔默认把放弃抢杠胡也并入过庄锁定,基准番数取该次放弃的番数;若你的玩法不锁抢杠胡,告诉我改回"仅点炮"〕
- 锁定:此后进入"过庄锁定态",**直到该玩家下次摸牌时解除**。
- 锁定期间:
  - **不能点炮胡**(别人打出能胡的牌也不能胡)
  - **可以自摸胡**(自摸不受过庄限制)
  - **例外解锁**:若锁定期间出现可点炮胡的牌,且**该次胡的番数 > 当初放弃那次的番数**,则可以胡(不受锁定)
- 计数:**每放弃一次重新计**(每次放弃都刷新锁定态与"放弃时番数"基准)。
- 引擎需为每个玩家维护:`passed_hu_lock`(是否锁定)、`passed_fan`(放弃时那手牌的番数)。

### 6. 牌局结束与惩罚结算
牌局有两种结束方式,处理**不同**:
- **提前结束**:未胡家数降到 **≤ 1**(即 ≥3 家已胡,含一炮多响一步胡到只剩 0~1 家未胡)。此时**不属于流局**:**不查叫、不查花猪、所有杠钱原样保留生效**,直接进入庄家轮转。
- **流局(荒庄)**:**牌墙摸完**时结束(此时必然还有 ≥2 家未胡,否则早已提前结束)。此时才执行下列惩罚结算。

流局(荒庄)的惩罚结算:
- **查大叫(查叫)**:对每个**未听牌**的玩家,赔付给每个**已听牌但未胡**的玩家,赔付额 = 该听牌者**理论可胡的最大番型对应分数**(枚举其听牌手牌所有可能补全的胡法,取最大番;只算**牌型结构番**——平胡/对对/金钩钓/七对/清一色/根——**不含**自摸、杠上花炮、海底、抢杠这类临场情境番)。
- **花猪并入查叫**:**仍有缺门牌的玩家(花猪)必然属于"未听牌",直接按查叫规则赔付即可,不设独立的花猪罚**。因为有定缺打法,被吃花猪概率极小;真出现时,花猪玩家就是一个普通的未听牌赔付方,走上面的查叫赔付,**无需** `flower_pig_penalty` 这类额外配置或加重惩罚。

- **验收**:典型听牌局面列出的听牌集合正确;最大番计算正确(构造一手能听多张、不同张番数不同的牌验算)

### 任务 6:状态、动作、杠、主引擎 `state.py`/`actions.py`/`gang.py`/`game.py`
- `state.py`:完整牌局状态——各家手牌(自己可见全部,对他人按信息可见性)、各家副露、牌河、牌墙剩余、当前轮到谁、**当前庄家**、各家是否已胡、**本局胡牌顺序(按结算发生顺序记录每位胡牌者)、首胡者、是否首胡即一炮多响、首炮多响放炮者/最近放炮者**(供庄家轮转与回放审计)、**各家过庄锁定态与放弃番数**、当前可触发的杠上花炮/抢杠胡/**海底(牌墙是否只剩最后一张/已摸最后一张)**标记、**各杠的完整流水记录(杠ID/杠者/类型/是否有钱/收取总额/逐笔付款人和金额/是否已转水/是否已退回/第几手杠/是否仍处于杠后待打转水窗口——供连续杠、杠转水与流局退杠钱追溯)**、开局阶段标记(换三张 / 定缺 / 对局)、本局换牌方向等
- `actions.py`:动作分**两类阶段**——
  - **开局动作**:`swap_three(3张同花色)` 换三张选择;`declare_void(花色)` 定缺选择
  - **对局动作**:打某张、碰、明杠、补杠、暗杠、胡、**抢杠胡**、过
  - `legal_actions(player)` **按当前阶段返回对应类型的合法动作**:换三张阶段只允许合法的同花色3张组合;定缺阶段只允许选一门;对局阶段才有摸打碰杠胡,**并正确施加定缺约束、过庄约束与海底最后一张禁杠约束**
  - **补杠的动作判定顺序**:玩家发起补杠 → 引擎先检查其他玩家可否抢杠胡(其合法动作含"抢杠胡")→ 有人抢则补杠作废转胡牌(+1番);无人抢补杠成立(按 4.1 判有无杠钱)
  - **打牌后的点炮检查**:玩家打出一张牌 → 引擎检查**其他所有未胡玩家**能否用它点炮胡 → 可能多家同时可胡(一炮多响),逐家给出"胡"的合法动作;海底/杠后等彩头番标记一并传入算番;若有玩家选择过,按过庄记录其放弃番数并继续收集其他可胡玩家响应
- `gang.py`:三种杠的判定与即时结算(直明杠被杠者付 2、暗杠每家付 2、**补杠无杠钱**);**处理抢杠胡流程;海底最后一张禁杠;连续杠各自即时结算并各自写流水;连续杠后只保留最后一次"有钱杠"(直明杠/暗杠)作为杠上炮/杠转水候选,补杠不作转水候选**
- `game.py`:`reset()` 执行完整开局(**发牌 → 掷骰定换牌方向 → 收集三家换三张选择并同步交换 → 收集定缺 → 庄家首打**);`step(action)` 推进(能处理开局动作与对局动作,含抢杠胡的优先判定、一炮多响的多家响应与逐个结算);杠成立后设置“杠后紧接弃牌”上下文,连续杠时用最新杠覆盖该上下文,若该弃牌点炮则触发杠上炮/杠转水;`legal_actions()`;`state` 访问;`run(policies)` 用给定策略跑完整局并返回每家最终得分;**每局结束按 4.7 确定下一局庄家**
- **验收**:能全自动跑完整局不崩、不出非法状态;开局换三张按骰子方向正确交换、强制同花色3张;定缺约束与过庄约束在 legal_actions 中正确体现(构造"放弃点炮后被锁、自摸仍可、更大番可解锁"的场景测试);掷骰方向表 11 个点数逐一测试正确;**补杠不产生任何杠钱(只改牌形/成根)、直明杠被杠者付 2、暗杠每家付 2;抢杠胡能正确抢断补杠并+1番结算;海底最后一张禁杠;连续杠每次独立结算,且杠上炮/转水只按最后一次杠处理;一炮多响多家正确逐个结算,杠后多响时每人各得杠上炮+1番且每人各得一份完整转水钱;胡牌顺序、首胡多响标记、放炮者记录正确;庄家轮转(首胡当庄/首炮多响放炮者当庄/流局连庄)正确**

### 任务 7:结算 + 验证套件 `settlement.py` + 整局测试(地基的验收关)
- `settlement.py`:单次胡牌结算(谁付谁、自摸全付/点炮单付)+ **一炮多响逐个胡牌者独立结算** + 杠即时分(补杠无钱)+ **杠转水(杠后点炮时把最后一次有钱杠——直明杠/暗杠——的全部杠钱转给胡牌者,与杠上炮+1番同时;多响时每位胡牌者各获得一份完整转水钱)** + 荒庄时查叫(花猪并入查叫)+ **荒庄时未听牌者杠钱原路退回(含回退其已发生的转水)**
  - **结算顺序约定**:
    1. 杠成立时按直明杠(被杠者付 2)/暗杠(每家付 2)执行即时零和结算,并写入杠钱流水;**补杠一律不产生杠钱、不写有钱流水**(只改牌形/成根)。
    2. 若杠后补摸又继续杠,允许连续杠;每次有钱杠各自即时结算、各自写流水,但“杠后点炮/转水候选”始终只指向最后一次有钱杠。
    3. 杠后补摸并紧接弃牌时,若该弃牌点炮,对每个胡牌者逐个计算点炮分,胡牌方式同时带 `after_kong=True` 计杠上炮 +1 番(**杠上炮是番,任何杠——含补杠——都算**);**仅当该次杠是有钱杠(直明杠/暗杠)时,再对其执行杠转水;若是补杠则无钱可转,只计杠上炮番**。
    4. 杠后点炮且一炮多响时,杠上炮 +1 番对每个胡牌者都生效;转水金额采用复制式而非平分:每个胡牌者各获得一份完整杠钱,杠者按胡牌人数吐出多份。
    5. 杠后紧接弃牌未点炮,或进入下一名玩家摸打/响应链后,该次杠转水窗口关闭;之后再点炮不追溯转水。
    6. 普通一炮多响时,放炮者分别向每个胡牌者支付其独立算出的点炮分;胡牌顺序按响应结算顺序写入 `win_order`,若这是本局第一次胡且多家同响,标记 `first_win_multi=true` 并记录该次放炮者。
    7. 牌墙摸完的荒庄时,先判定每个未胡玩家是否听牌;未听牌玩家作为杠者收取过的所有杠钱全部按原付款人、原金额退回,**若其杠曾转水给某胡牌者,该转水也一并作废、从该胡牌者账上扣回再原路退给付款人**;已听牌玩家收取的杠钱保留。提前结束(≥3 家胡)不做此退款,杠钱全保留。
    8. 杠钱失效/退回后,再执行查叫(花猪作为未听牌的一种并入查叫,无独立花猪罚);整个过程每一步都必须保持零和。
    9. 本局结束后计算下一局庄家:本局有人胡且首胡不是多响 → 首胡者当庄;首胡即一炮多响 → 该次放炮者当庄;全程无人胡的荒庄 → 原庄连庄。
  - **杠钱流水模型建议**:每条有钱杠记录包含 `gang_id`、`gang_player`、`gang_type`、`payments[(payer, amount)]`、`total_amount`、`hand_step`、`transferable_until_discard_resolved`、`transferred_to`、`transfer_count`、`refunded`。杠转水不是删除原记录,而是在分数上追加“杠者→胡牌者”的转水流水;多响时追加多笔同额转水流水,并标记转水次数,便于回放和流局审计。
- **整局守恒检查**:每局结束,所有玩家得分总和必须 = 0(零和校验,抓结算 bug 的利器;**杠转水、流局退杠钱这类复杂结算后零和仍须严格成立**,这是验证它们正确的关键)
- **专项测试**:构造"有钱杠后打牌点炮"场景,验证杠钱全额转给胡牌者 + 杠上炮+1番同时生效;构造"补杠后点炮"场景,验证只计杠上炮+1番、无转水;构造"荒庄且杠者未听牌、且该杠中途已转水给某胡牌者"场景,验证杠钱原路退回、转水也从受让胡牌者处扣回作废;各场景结算后零和守恒
- **已知牌局回归**:构造若干"从发牌到结束、每步动作、各家应得分"都人工算好的完整对局,引擎跑出来必须逐分对上
- **验收**:批量随机跑 10000 局零和校验全过(总和恒为0);已知牌局逐分对上;无崩溃无非法状态

## 七、本阶段成功的标志

不是"能打牌",而是:**10000 局随机对局零和守恒恒成立 + 所有番型组合算分经手工验算正确 + 过庄/查叫/花猪等特殊规则在测试中逐条验证通过**。引擎算分一旦可信,它就成了合格的"练习场 + 裁判",S3/S4/S5 才有正确的地基。
