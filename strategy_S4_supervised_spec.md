# S4 需求规格:监督学习(belief 网络 + 模仿策略冷启动)

> 策略 AI 项目第四阶段。S1 引擎、S2 状态层(s2.v4)、S3 规则基线均已定稿验证通过,S3 已产出"协议状态 + 动作 + 上帝视角标签"格式的自对局数据。
> 本阶段是**第一次基于数据建模**:训练两个神经网络——① belief 网络(读牌:从残缺观测推断未知牌分布)② 模仿策略网络(模仿 S3,作为 S5 强化学习的冷启动)。
> 交给 AI 编码助手执行。Python 3.10+,PyTorch,依赖少,有单测。

---

## 设计定位(给执行者的关键认知)

**S4 的目标不是"打得比 S3 强"。** 模仿学习的天花板就是老师(S3),这是原理性的,不要试图在 S4 里超越。S4 的真正交付是四样东西:

1. **belief 网络**:整条线里第一个"学习式读牌"能力——输入残缺观测,输出每张未知牌在牌墙/各对手手中的概率。这是辅助模型在真实环境(信息残缺、有噪声)下决策质量的核心来源,验收标准是**显著优于 S2 的 `PriorBelief` 先验回退且校准良好**;
2. **模仿策略网络**:一个"会打牌的神经网络",让 S5 不用从随机权重开始瞎撞,直接从"S3 水平"起步;
3. **训练基础设施**:数据管线(退化 → 编码 → 批量化)、训练循环、评估竞技场——S5 全部复用;
4. **对 S2/S3 交付质量的实战检验**:数据格式、编码器、退化算子、标签管线,任何一环有问题都会在本阶段暴露。

**两条铁律(继承自 S2 v4,贯穿本阶段)**:
- **上帝视角只进标签,永不进输入**。belief 网络的输入是玩家视角的退化观测,标签才是真实牌位;策略网络的输入同样不含任何隐藏信息;
- **训练观测必须退化**。所有网络都在观测退化后的残缺观测上训练(混入部分完美观测做课程式过渡),否则部署时遇到中途接入/识别噪声会断崖失效。

## 一、两个网络的职责与关系

### 1. belief 网络(先训)

- **输入**:玩家视角、经观测退化的 s2.v4 编码(facts + statistics 区段;**不含** beliefs 区段——它自己就是 beliefs 的生产者)
- **输出**(与 S2 `TileBelief` 接口对齐):
  - `tile_location_beliefs`:27 种牌 × {牌墙, 下家, 对家, 上家} 的概率分布(满足行归一;与已知的未知区分解守恒兼容)
  - `opponent_tenpai_beliefs`:各未胡对手的听牌概率
  - `discard_danger`:每张候选弃牌对每个对手的点炮概率
- **标签**(来自 S3 数据记录的上帝视角标签):每张未知牌的真实位置、各对手真实听牌状态、各候选弃牌是否真实点炮张
- **损失**:牌位分布用逐牌交叉熵(按未知区掩码,已现牌不参与);听牌/点炮用二元交叉熵
- **训练配方的关键**:输入是退化观测、标签是精确真值——网络由此学会"从残缺信息推断",这正是产品的核心能力

### 2. 模仿策略网络(后训)

- **输入**:玩家视角、经观测退化的 s2.v4 完整编码,其中 beliefs 区段由**训练好并冻结的 belief 网络**(以 `LearnedBelief` 身份接入 S2 接口)填充
- **输出**:动作空间上的概率分布;**logits 必须先套 legal mask 再归一**(非法动作概率恒为 0)
- **标签**:S3 在该决策点选择的动作(交叉熵)
- **两阶段而非端到端**:先训 belief、冻结、再训策略。理由:① belief 有独立的真值标签,单独训更快更稳、可单独验收;② 接口清晰,S5 可以只微调策略而保持读牌能力;③ 出问题时能定位是"读牌不准"还是"决策不好"。端到端联合微调留给 S5 视情况决定

### 3. 网络结构约束(刻意保守)

- 首选简单结构:编码向量 → MLP(残差块)双头/多头输出。**不要一上来就上 Transformer/序列模型**——先让简单模型在管线上全部跑通、指标达标,结构升级是后续优化项,且必须以"显著优于当前简单版"为升级理由
- 模型要小:单次推理毫秒级(CPU 可接受),因为 S5 自博弈要海量调用、部署端(视觉辅助场景)也未必有强 GPU
- 所有随机性受种子控制,训练可复现

## 三、任务清单

### 任务 1:数据管线 `dataset_builder.py` + `splits.py`

**做什么**:把 S3 记录的协议状态,在线退化 + 编码成训练批次。

- 读取 S3 数据记录(协议 JSON + 动作 + 标签),**按局**划分 train/val/test(同一局的决策点高度相关,按决策点划分会造成评估虚高;划分比例可配,默认 90/5/5)
- **退化在线做**(每个 epoch 对同一决策点采样不同退化,等效于数据增广):调用 S2 `observation_degradation`,退化配置可指定"画像混合"——完美观测 / 轻度噪声 / 中途接入 / 重度残缺按比例混合,支持课程式调度(前期完美比例高,后期残缺比例高)
- 编码调用 S2 `encoder.py`;belief 训练时输出"编码(不含 beliefs 区段)+ 标签",策略训练时输出"完整编码 + 动作标签 + legal mask"
- 数据量:S3 侧按需补跑自对局,目标 **≥5 万局**(可配;数据不够时先跑通小规模再扩量)
- **验收**:同种子同输出;吞吐能喂饱训练(给出样本/秒数字);抽样断言输入侧无任何上帝视角字段;划分无局间泄漏(同一局 ID 只出现在一个 split);退化画像比例统计与配置一致

### 任务 2:belief 网络 `belief_net.py` + `train_belief.py`(本阶段核心)

**做什么**:实现并训练读牌网络。

- 结构:MLP + 残差块,三个输出头(牌位分布 / 听牌 / 点炮),头部结构与 S2 `TileBelief` 接口输出对齐
- 训练循环:配置驱动(学习率、批大小、退化画像、早停);checkpoint 带数据版本与配置指纹;训练曲线落盘(CSV 即可,不强制引入实验管理框架)
- `metrics.py` 实现:逐牌 log-loss、Brier 分数、ECE(期望校准误差)、按局面阶段(开局/中盘/尾盘)与退化画像分桶的指标
- **验收(必须全部给出数字)**:
  1. held-out 测试集上,牌位分布 log-loss **显著优于 `PriorBelief`**(这是硬门槛——先验回退是免费的,学习版必须证明自己值得);
  2. **尾盘提升幅度最大**:按阶段分桶后,尾盘桶的相对提升应明显大于开局桶(尾盘信息多、可推断空间大,这是"高手数牌"直觉的验证点;若尾盘反而不如先验,说明训练有 bug);
  3. 校准:ECE 低于阈值(建议 ≤0.05,可调),并输出可靠性图数据——概率说 70% 的事就应该约 70% 发生,**辅助模型给人看的概率必须诚实**;
  4. 退化鲁棒:各退化画像(含中途接入)下均优于同画像的 `PriorBelief`;
  5. 可见性红线回归:构造隐藏牌不同、可见信息相同的状态对,网络输入特征必须逐位相同

### 任务 3:接入 S2 接口 `LearnedBelief`

**做什么**:把训练好的 belief 网络包装成 S2 `TileBelief` 接口的 `LearnedBelief` 实现,替换 `PriorBelief` 后整条 S2 链路照常运行。

- 加载 checkpoint、推理、输出格式与接口约定一致(归一/守恒约束在输出侧强制:与 `tile_counting` 的未知区分解调和)
- 推理确定性(eval 模式、固定种子);批量推理支持(S5 自博弈需要)
- **验收**:S2 的接口互换测试全过;from_engine → 统计 → LearnedBelief → 编码 → legal mask 全链路 100 局无错;单点推理延迟与批量吞吐给出数字

### 任务 4:模仿策略网络 `policy_net.py` + `train_policy.py`

**做什么**:训练模仿 S3 的策略网络。

- 输入含 `LearnedBelief` 填充的 beliefs 区段;logits 套 legal mask;交叉熵训练
- 开局动作(换三张/定缺)与对局动作可分头或分模型处理(S3 数据里已分开标记),取实现简单者
- **验收(必须全部给出数字)**:
  1. held-out 动作一致率(top-1 与 S3 相同)达标:完美观测下建议 ≥85%(阈值可按实测调整,但须报告);
  2. **在线实战**:arena 里"1 个策略网络 + 3 个 S3"跑 ≥10000 局——全程**零非法动作**、零和守恒;平均得分与"4 个 S3 互打"的基线差距在容差内(模仿不要求赢过老师,但不能明显更差);
  3. **残缺观测赛道**:中途接入画像下同样实战 ≥1000 局,行为不崩(零非法动作、得分下降幅度在报告中量化)——这是部署场景的预演;
  4. 退化训练有效性对照:用"只在完美观测上训练"的对照版策略在残缺赛道上对比,验证退化训练确实带来鲁棒性提升(这是对 v4 核心假设的一次实证,数字写进报告)

### 任务 5:评估竞技场 `arena.py` + 评估报告

**做什么**:标准化的对局评估工具,S5 直接复用。

- 任意策略组合(随机/贪心/S3/策略网络/未来的 S5 网络)按指定座次与局数对打;统计得分分布、胡牌率、流局率、番型分布、置信区间
- 支持退化观测赛道:对指定座位的玩家施加观测退化(其余座位完美观测),模拟"辅助模型在残缺信息下 vs 正常对手"
- 产出一份 S4 评估报告(markdown):任务 2/4 的全部验收数字、分桶表、对照实验结论
- **验收**:arena 结果可复现(同种子同结果);报告数字齐全

### 任务 6:知识注解 `docs/concepts.md`

- 监督学习与模仿学习是什么;为什么模仿的天花板是老师、S4 却仍然必要(冷启动 + 基础设施 + belief)
- belief 网络为什么单独训、单独验收;"输入退化观测 + 标签精确真值"这个配方为什么能教会网络从残缺信息推断
- 校准(calibration)是什么;为什么辅助模型输出的概率必须校准(给人看的 70% 必须是真 70%)
- train/val/test 为什么按局划分;过拟合与早停
- **验收**:学习者读完能说清 belief 网络和策略网络各自解决什么问题、S5 从哪里接手

## 四、全局技术约束

- Python 3.10+;PyTorch;其余依赖尽量少(numpy、pytest;不强制引入 lightning/wandb 等框架)
- **上帝视角只进标签**:数据管线、模型输入、评估输入全程可见性无泄漏,单测死守(继承 S2 红线)
- **退化训练是默认**:任何"只在完美观测上训练"的产物只能作为对照实验存在,不得作为交付物
- 模型小、CPU 可推理;单点推理毫秒级(报告实测数字);checkpoint 自带配置与数据指纹,可精确复现
- 同种子全流程可复现(数据划分、退化采样、初始化、训练)
- 每模块 pytest 单测;数据管线的无泄漏与无局间泄漏、belief 校准、策略零非法动作是重点测试对象

## 七、本阶段成功标志

**belief 网络在 held-out 与全部退化画像上显著优于先验回退、校准达标、尾盘提升最大;以 `LearnedBelief` 身份无缝接入 S2 链路;模仿策略网络零非法动作、实战接近 S3、在中途接入等残缺赛道上行为不崩;退化训练的有效性有对照实验证据;竞技场与训练基础设施可被 S5 直接复用。**

达标后,S5 拿到的是:一个会读牌的 belief 网络 + 一个 S3 水平的起点策略 + 一套现成的训练评估设施。
